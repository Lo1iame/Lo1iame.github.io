组件传值

Vue开发中，我们经常遇到需要组件间传值的情景

父组件传给子组件

在传统的实现方式中，父组件传递给子组件使用Prop

<!-- 父组件 -->

<child msg="来自父组件的值"></child>
// 子组件
props: {
    msg: String,
}

// TypeScript
@prop() msg: string

子组件传给父组件

子组件传递给父组件使用emit

<!-- 父组件 -->

<!-- 监听子组件的sendMsg事件 -->
<child @sendMsg="getMsg"></child>
// 父组件

// 响应事件函数
getMsg(msg) {
    console.log(msg)
}
// 子组件

this.$emit('sendMsg', '传给父组件的消息')

消息总线

如果我们遇到兄弟组件，或者沾不上什么亲的随便两个组件需要互相响应，可以通过 消息总线(Bus) 来实现。

消息总线的原理

Vue实例中有$on()和$emit()方法，我们可以用一个新的Vue实例作为消息总线，让它集中管理消息的发送和接收响应。

实现方式

TypeScript

// Bus.ts

import Vue from 'vue';

/**
 * @class 消息总线
 */
export class Bus {

    // 消息处理者
    private handler = new Vue;

    /** 
     * @description 传出消息
     * @param event 消息文本
     * @param preload 载荷
     */
    $emit(event: string, preload?: any): void {
        this.handler.$emit(event, preload);
    }

    /** 
     * @description 响应消息
     * @param event 消息文本
     * @param handle 响应消息回调函数
     */
    $on(event: string, handle: Function): void{
        this.handler.$off(event);   // 先解绑原有的事件，防止多次执行回调
        this.handler.$on(event, handle);
    }
}

JavaScript 按此逻辑封装即可

Bus类有一个处理器handler，是一个新的Vue实例，同时有两个方法$on()和$emit()，负责响应消息和传出消息

使用

// main.ts

import { Bus } from './bus/Bus';
Vue.prototype.$bus = new Bus;
// 组件A
this.$bus.$emit('event name', '消息内容');
// 组件B
this.$bus.$on('event name', (msg)=> {
    console.log(msg); // 消息内容
})
